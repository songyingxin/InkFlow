# InkFlow 优化方案（v4）

> 制定时间：2026-06-24  
> **需求 SSOT：** [业务逻辑/00-作者需求逻辑.md](业务逻辑/00-作者需求逻辑.md)  
> 原则：**写/存仅正文**；**章摘要 + 设定 = 前端按钮 + daily_sync 直调工具**；**不注册 Updator Subagent**；**未来细纲显式请求**

## 〇、作者需求摘要

| 作者要什么 | 技术对应 |
|------------|----------|
| 写章不被 sync / 改细纲打断 | 写章路径仅保存正文 |
| 历史大纲跟正文 | **[同步设定]** batch 内 `update_chapter_summaries` |
| 设定自己对一下 | Editor 顶栏 **[同步设定]** 按钮 → `run_daily_sync()` |
| 可选提醒 | 当日首次打开轻提示，与按钮同一 API |
| 未来规划我自己定 | 独立入口「同步细纲」；不在同步设定 batch 内 |
| 聊天只感知写 / 改 / 问 | **4 个 Subagent**：Reader / Creator / Editor / Critic |

## 一、核心设计：维护三层 + 入口模型

```
Tier A · 正文写入（写/存即落盘）
  └── chapters/NNN.md + outline is_written / content_hash
  ❌ 不写章后 LLM 摘要；不编辑器保存后自动摘要

Tier A′ · 摘要 + 设定归档（用户点按钮，非 Subagent）
  └── [同步设定] 按钮 → daily_sync → update_chapter_summaries → sync_* / scan_*
  └── 可选：当日首次打开轻提示（同一 run API）
  ❌ 不写章后自动 sync；不 handoff Updator
  ✅ 删章：outline 条目（含 content_summary）直接移除

Tier B · 规划（用户主动）
  └── update_outline / generate_outline / update_field

Tier C · 补救
  └── 缺摘要补漏；Chat「同步设定」快路由（同 pipeline）
```

### 为什么不用 Updator Subagent

| 维度 | Subagent | 按钮 + daily_sync |
|------|----------|-------------------|
| 用户认知 | 第五个助手 | 像保存/导出 |
| 延迟 | Lead 路由 + ReAct | 点即跑 |
| 任务性质 | 固定 pipeline | 固定 pipeline |

**Updator 仅作维护模块命名**（`agent/maintenance/`），**不**出现在 `AGENT_REGISTRY`。

### Subagent 列表（4 个）

| Subagent | 用户感知 |
|----------|----------|
| Creator | 写 |
| Editor | 改 |
| Reader | 问 |
| Critic | 无（后台） |

## 二、设定同步：产品行为

### 2.1 主入口 — Editor 顶栏按钮

**位置：** `EditorPage` 顶栏 / 工具栏，与保存并列。

**状态展示：**

- 无待同步：按钮可用，文案「同步设定」
- 有待同步：角标「N 章待同步」或 secondary 强调
- 上次同步：`上次同步：今天 8:12`（tooltip 或设置区）

**点击流程：**

1. 确认弹窗：  
   「将根据第 X–Y 章更新：角色、关系、伏笔、写作设定。**不会修改未来细纲。**」  
   [开始同步] [取消]
2. `POST /api/maintenance/daily-sync/run` → SSE 进度 + 字段 diff
3. 完成 → toast「设定已同步」；无变化 →「设定已是最新」

### 2.2 辅助 — 可选每日提醒

满足条件时**轻提示**（不挡写作）：

- 当日未 dismiss；有待同步章节；可选 `daily_sync_prompt_hour ≥ 8`

> 有 3 章未同步到设定。[立即同步] [今天不再提醒]

「立即同步」= 与按钮同一 `run` API。按钮始终可见，跳过提醒仍可手动点。

### 2.3 Chat 补救（非主路径）

用户说「同步设定」→ `intent` FastDirectTool 调 `run_daily_sync` 或逐步 `sync_*`。  
Lead **不** handoff 到 Updator（已移除）。

### 2.4 pipeline 顺序

见 `novel_agent/agent/maintenance/README.md`：

`sync_characters` → `sync_relationships` → `scan_foreshadowing` → `sync_settings`

**不含** `update_outline`。

### 2.5 meta 字段

```json
{
  "last_daily_sync_date": "2026-06-24",
  "daily_sync_dismissed_date": "2026-06-23",
  "daily_sync_prompt_hour": 8
}
```

## 四、分阶段实施

### 阶段 1 · P0（1–2 天）

- 1.1 ✅ 写/存仅正文；摘要进 daily_sync；删章移除摘要
- 1.2 `intent` 修正（sync → FastDirectTool；删「更新摘要」快路由）

### 阶段 2 · 同步按钮（2–3 天）✅ 已实现

**后端**

- `agent/maintenance/daily_sync.py` — `get_daily_sync_status` / `stream_daily_sync` / `dismiss_daily_sync_prompt`
- `api/routes/maintenance.py` — `GET status` / `POST run`（SSE）/ `POST dismiss`
- `MetaInfo` 增加 `last_daily_sync_date` 等字段

**验收：** 连写多章不 sync；点按钮后 pipeline 跑完；不写章钩子。

### 阶段 3 · 瘦身

### 阶段 4 · 工程债（按需）

ChatState 抽离、checkpointer、workspace gitignore、generate 返回摘要等。

## 六、明确不做

| 项 | 理由 |
|----|------|
| Updator Subagent | UX + 固定 pipeline，按钮更合适 |

| Chat 注入为主入口 | 按钮为主 |
