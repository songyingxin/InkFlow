# Novel-LangGraph 需求文档

## 1. 项目概述

基于 LLM 的网文写作 Agent，采用 ReAct Agentic Loop 架构，为网文作者提供从项目初始化到章节续写、大纲管理的全流程写作辅助。支持 Web 界面交互，右侧聊天区通过自然语言指令驱动 Agent 自动路由到对应工具执行，同时支持多轮对话引导。

## 4. 数据模型

### 4.1 记忆文件体系

```
workspace/<书名>/
├── config.md           ← 写作设定（风格、世界观、修炼体系、主角设定）
├── outline.md          ← 总大纲 + 各卷分纲（✅/📋 标记进度）
├── characters.md       ← 人物关系图谱（按势力分组）
├── summaries.jsonl     ← 所有章节摘要（JSONL，每行一条）
├── foreshadowing.md    ← 伏笔清单（活跃/回收/废弃分区）
├── meta.json           ← 元信息（书名、总章数、各字段已读章节数）
├── MEMORY.md           ← 长期记忆（提炼后的持久事实）
├── memory/             ← 每日日志目录
│   └── YY-MM-DD.md     ← 当天操作记录
├── chat.jsonl          ← 详细对话记录（parentId 链式结构）
└── chapters/           ← 章节全文（001.md, 002.md, ...）
```

### 4.2 三层记忆系统

```
┌─────────────────────────────────────────────────────────┐
│ 第一层：长期记忆 MEMORY.md                                │
│   提炼自 daily memory 的通用持久事实                       │
├─────────────────────────────────────────────────────────┤
│ 第二层：每日日志 memory/YY-MM-DD.md                       │
│   每隔 10 轮对话，LLM 从 chat.jsonl 提炼生成              │
│   跨天时自动压缩：分类提炼到字段文件或 MEMORY.md，删除原日志  │
├─────────────────────────────────────────────────────────┤
│ 第三层：对话记录 chat.jsonl                               │
│   原始消息，parentId 链式结构，支持对话树                   │
└─────────────────────────────────────────────────────────┘

提炼分发策略（跨天压缩时）：
  [大纲] → outline.md 末尾追加
  [设定] → config.md 末尾追加
  [人物] → characters.md 末尾追加
  [伏笔] → foreshadowing.md 末尾追加
  [通用] → MEMORY.md 追加
```

### 4.3 批量导入 JSON 格式

```json
[
  {
    "标题": "第一卷 第一章 星辰之始",
    "正文内容": "章节正文内容..."
  },
  {
    "标题": "第一卷 第二章 初入宗门",
    "正文内容": "章节正文内容..."
  }
]
```

### 4.4 核心状态模型

```
NovelState（顶层状态，全局单例）
├── outline: NovelOutline              ← 大纲结构（卷级 + 章节列表）
│     └── chapters: ChapterOutline[]   ← 各章节大纲条目
├── memory_files: MemoryFiles          ← 记忆文件路径配置（11 个路径）
├── meta: MetaInfo                     ← 元信息（书名、章数、各字段已读进度）
├── summaries: ChapterSummary[]        ← 章节摘要列表缓存
├── config_md_content                  ← 写作设定缓存
├── outline_md_content                 ← 大纲缓存
├── characters_md_content              ← 人物关系缓存
└── foreshadowing_md_content           ← 伏笔清单缓存

ChatState（LangGraph 工作流状态，每次对话创建）
├── messages: list[dict]               ← 对话消息列表（OpenAI 格式）
├── novel_state: NovelState            ← 引用全局小说状态
├── round_count: int                   ← 对话轮次计数
├── is_complete: bool                  ← 任务是否已完成
├── iteration: int                     ← agent 循环次数
├── reflexion: str                     ← 最新反思内容
├── tool_call_count: int               ← 累计工具调用次数
├── tool_results: list[str]            ← 本轮工具执行结果
├── user_request: str                  ← 用户原始请求
├── field_values: dict                 ← 前端传入的字段值
├── last_chat_entry_id: str            ← chat.jsonl 最后一条记录 ID
└── _stream_writer                     ← SSE 流写入器
```

## 5. API 端点

### 5.1 书籍管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/books` | 获取所有书籍列表 |
| POST | `/api/books/create` | 创建新书籍 |
| POST | `/api/books/select` | 选择当前书籍 |
| POST | `/api/books/delete` | 删除书籍 |

### 5.2 章节管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/chapter-content/{idx}` | 获取章节内容 |
| POST | `/api/chapter/add` | 添加章节 |
| DELETE | `/api/chapter/delete/{idx}` | 删除章节 |
| POST | `/api/chapter/import` | 导入单个章节 |
| POST | `/api/chapters/import-batch` | 批量导入章节 |
| POST | `/api/chapter/update/{idx}` | 更新章节 |

### 5.3 字段编辑

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/state` | 获取当前状态 |
| POST | `/api/update-field` | 更新字段内容 |

### 5.4 AI 生成（直接调用，绕过 Agent）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/generate-chapter-title` | 生成章节标题 |
| POST | `/api/generate-chapter-stream/{idx}` | 流式生成章节内容（已废弃） |
| POST | `/api/generate-field-stream` | 流式生成字段内容（已废弃） |

### 5.5 Agent 对话（主入口）

| 方法 | 路径 | 说明 |

## 6. Agent 架构

### 6.1 ReAct Agentic Loop

```
START → agent_node ─┐
         ↑           ├→ [完成] → memory_update_node → END
         └── [继续] ─┘

agent_node 内部循环（Claude Code 模式）：
  while LLM 返回 tool_calls:
    execute tools → append results → call LLM again
  LLM 返回纯文本 → 判断任务是否完成

任务完成判断：
  主路径：模型调用 task_complete 工具 → 显式标记完成
  兜底策略：
    - 纯对话（本轮无工具调用）→ 完成
    - 本轮工具结果含"失败" → 未完成，重试
    - 达到 MAX_ITERATIONS(5) → 强制完成
```

### 6.2 Agent 工具清单

| 工具名称 | 功能 | 触发条件 |
|---------|------|---------|
| task_complete | 标记任务完成 | Agent 认为所有操作已执行完毕 |
| continue_writing | 续写/生成新章节 | 「续写」「写下一章」「生成新章节」 |
| regenerate_chapter | 重新生成指定章节 | 「重写第N章」「重新生成」 |
| generate_outline | 生成/更新大纲 | 「生成大纲」「更新大纲」「调整大纲」 |
| generate_config | 生成/更新写作设定 | 「生成设定」「更新设定」「完善世界观」 |
| generate_characters | 生成/更新人物关系 | 「更新人物」「人物关系」「梳理角色」 |
| generate_foreshadowing | 生成/更新伏笔清单 | 「更新伏笔」「伏笔管理」「追踪伏笔」 |

### 6.3 记忆更新节点

```
memory_update_node 在每次任务完成后执行：
  1. 跨天压缩：检查旧日志 → LLM 提炼分类 → 分发到字段文件/MEMORY.md → 删除原日志
  2. 定期日志：每 10 轮对话 → LLM 从 chat.jsonl 生成日志 → 追加到 daily memory
```

### 6.4 消息压缩机制

```
当对话消息超过 MAX_MESSAGES_BEFORE_COMPACT(40) 条时：
  保留最近 20 条 + 系统提示 + 压缩摘要（LLM 生成）
  避免上下文窗口溢出
```
