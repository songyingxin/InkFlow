## 1. 这篇管什么

| | |
|--|--|
| **作者想干什么** | 「我这几天写了不少，帮系统记住书里的新事实」 |
| **入口** | **Editor 顶栏 [同步设定]**（主路径） |
| **铁律** | 不改未来细纲；不是 Chat 里第五个助手 |

**按钮含义：** 把最近写的收进系统记忆 = 补章摘要 + 同步角色 + 地点 + 关系 + 伏笔 + 写作设定。

Chat 里说「同步设定 / 补摘要 / 扫描伏笔」→ **不跑 batch**，只弹引导让用户点按钮。

---

## 2. 点按钮后跑什么（`daily_sync`）

固定顺序，一步接一步：

| 步骤 | 做什么 | 代码 |
|------|--------|------|
| 1 | 给缺摘要或正文已变的章补摘要 | `update_chapter_summaries` |
| 2 | 从未读章正文增量更新角色表 | `sync_characters` |
| 3 | 增量更新地点表 | `sync_locations` |
| 4 | 增量更新关系网 | `sync_relationships` |
| 5 | 扫描伏笔变化 | `scan_foreshadowing` |
| 6 | 增量更新写作设定（宏观） | `sync_settings` |

**重要：** `update_chapter_summaries` **只能**出现在上面的 batch 里，写章、保存、同步细纲时都不能偷偷调用。

**什么时候角标亮（有待同步）：**

- 有已写章还没有摘要，或
- 任一档案的 read 指针落后于最新已写章号

## 3. API

| 接口 | 用途 |
|------|------|
| `GET /api/maintenance/daily-sync/status` | 角标、范围、是否弹当日提示 |
| `POST /api/maintenance/daily-sync/run` | SSE 流式跑完整 batch |
| `POST /api/maintenance/daily-sync/dismiss` | 关掉当日横幅 |

## 7. 自测

| 操作 | 预期 |
|------|------|
| 点同步设定 | SSE 六步跑完；摘要 + 地点 + 各档案更新 |
| Chat「同步设定」 | 不调 handler；UI 引导点按钮 |
| GET status | 缺摘要或 read 落后时 pending=true |
| 同步后再续写 / 提问 | 能用到最新摘要和档案 |
