## 1. 这篇管什么

| | |
|--|--|
| **作者想干什么** | 改名、加规则；「后面几章打算怎么写」；重做未来细纲 |
| **入口** | Chat · Editor；侧边栏直接改 md |
| **铁律** | 不覆盖已写正文；改完可能提醒「建议再看看关系网 / 细纲」，但不强制全自动改 |

**和 [同步设定] 的分界：** 本文只动 **未来细纲（打算）**。章摘要和角色/地点等档案的 batch 更新 → [01-对](01-对-功能设计.md)。

**和「写」的分界：** 书还是空的、整份生成设定 → [01-写](01-写-功能设计.md) · `generate_*`。

## 2. 功能一览

| 作者怎么说 | 做什么 | 代码 | 说明 |
|------------|--------|------|------|
| 把主角改名 / 加一条世界观规则 | 局部改某个字段 | `update_field` | 改 settings、characters、locations 等 |
| 同步细纲 / 更新未来大纲 | 按已写内容增量更新细纲 | `update_outline` | 只动 `outline_future.md` |
| 重新生成未来细纲 | 整份重做细纲 | `generate_outline` | **已有书**时用；新书冷启动见写册 |

**局部改之后的提醒（产品行为，不是自动写）：** 改了 settings / characters / locations 等，在回复里提示「建议检查关系网、未来细纲、伏笔表」。

**同步细纲的前提：** 需要已有章摘要来拼 historical outline。摘要缺了应说「请先点同步设定」，**不能**在 `update_outline` 里偷偷补摘要（当前实现有这个问题，待修）。

## 3. 入口

### Chat

| 作者大概会说 | 调什么 |
|--------------|--------|
| 同步细纲 / 更新未来大纲 | `update_outline` |
| 重新生成未来细纲 | `generate_outline` |
| 改名 / 改一条规则 | `update_field` |

路由见 [索引 §7](01-意图分册索引.md)。

### Editor（不经过 Chat）

| 操作 | 说明 |
|------|------|
| 侧边栏直接编辑字段 md | 等同手动改文件，不调 LLM |

## 5. 自测

| 操作 | 预期 |
|------|------|
| Chat「同步细纲」 | 只更新 outline_future；不动四档案 |
| Chat「重新生成未来细纲」 | 整份重写 outline_future |
| 侧边栏改 characters.md | 直写文件，无 LLM |
| 缺章摘要时同步细纲 | 阻断或提示，不偷偷补摘要 |
