## 2. 两个必须分开的动作

| | **[同步设定]**（对） | **同步细纲**（改） |
|--|---------------------|-------------------|
| **作者心里的问题** | 「最近写的，系统记住了吗？」 | 「后面几章打算怎么写？」 |
| **会改什么** | 章摘要 + 角色 + 地点 + 关系 + 伏笔 + 写作设定 | 未来细纲 |
| **性质** | 从正文提取**已发生的事实** | **创作规划**（可随时推翻） |
| **入口** | Editor 顶栏按钮 | Chat：「同步细纲」「更新未来大纲」 |
| **典型时机** | 连写一批后、每天一次 | 改走向、卡文、开新卷 |

开新卷对账：先 **同步设定** → 再 **同步细纲**。顺序自定，**不能绑在同一按钮**。

---

## 3. 铁律（改代码前先看）

| 规则 | 说明 |
|------|------|
| 写章 / 保存只动正文 | 不当场生成摘要、不 sync 档案、不改细纲 |
| 章摘要只在「同步设定」里生成 | `update_chapter_summaries` 只能在 `daily_sync` 里调用 |
| Chat 说「同步设定」不跑 batch | 返回 UI 引导，让用户点顶栏按钮 |
| 同步细纲不偷偷补摘要 | 缺摘要时应提示「请先同步设定」，不在 `update_outline` 里静默补 |
| 已写正文是最高真相 | 系统不得擅自覆盖（除非用户明确「重写第 N 章」） |

## 4. 跨意图分工

| 作者想做什么 | 归哪一册 | 代码入口（供查实现） |
|--------------|----------|----------------------|
| 续写 / 重写 / 保存正文 | 写 | `continue_writing` · `regenerate_chapter` · `chapter_service.update` |
| 开新书 / 字段还是空的时「生成设定/角色/…」 | 写 | `init_novel` · `generate_*` |
| 已有书「同步细纲 / 重做未来细纲」 | 改 | `update_outline` · `generate_outline` |
| 已有书局部改名、加规则 | 改 | `update_field` |
| 正文沉淀进摘要和档案 | 对 | `daily_sync`（内含 6 步 sync） |
| 问伏笔 / 节奏 / 一致性 | 问 | `read_novel_content` · `check_consistency` 等 |
| Chat「同步设定 / 补摘要 / 扫描伏笔」 | 对（仅引导） | `guide_sync_button`，不直调 handler |
| 删章 | 写 | `delete_chapter`（摘要条目一并删，不调 LLM） |

**三种更新档案的方式，别混：**

```
空书整字段生成     → 写    generate_*        整份重写
已有书改一条       → 改    update_field      局部 patch
正文写完后沉淀     → 对    daily_sync          增量 sync + 摘要
已有书调后面规划   → 改    update_outline 等
```

## 5. 数据怎么流（全局）

```
开新书 init_novel
      │
      ▼
可选：生成设定 / 角色 / 细纲 …（写）
      │
      ▼
续写、保存 → chapters/*.md（正文，唯一真相）
      │
      │  保存时若正文变了 → 该章旧摘要作废，等下次同步
      ▼
作者点 [同步设定] → daily_sync
      │
      ├── 补章摘要
      └── 同步角色 / 地点 / 关系 / 伏笔 / 设定
      │
      ▼
问（查记忆）· 续写（读上下文）变准
      │
改 · 同步细纲（读 historical outline，依赖上面补好的摘要）
```

档案若落后于最新已写章：续写/问答仍可跑，但质量下降；顶栏角标提示待同步。

## 6. Chat 助手与意图

| 助手 | 对应意图 | 收敛方向 |
|------|----------|----------|
| **Creator** | 写 | 续写已走快速路由；generate_* 待直调 |
| **Reader** | 问 | 简单问句待直调，复杂仍多轮 |
| **Editor** | 改 | 细纲 / 改字段待直调 |
| （无第五个助手） | 对 | 只有 Editor **按钮** + API |

续写默认不跑 Critic，避免拖慢心流。

## 7. Chat 怎么说 → 走哪条路

> 实现：`novel_agent/agent/multi_agent/intent.py`

| 作者大概会说 | 怎么走 |
|--------------|--------|
| 续写 / 写下一章 | 直调 `continue_writing` |
| 重写第 N 章 | 直调 `regenerate_chapter` |
| 创建新书 | Creator → `init_novel` |
| 生成设定 / 角色 / 关系 / 伏笔 / 地点 / 大纲（冷启动或整份重做） | Creator → `generate_*` |
| 重新生成未来细纲 | Editor → `generate_outline`（与写册同一 handler，属**改**意图） |
| 检查一致性 / 伏笔在哪 | 待直调 `check_consistency` 等 |
| 复杂、含糊的问句 | Reader 多轮 |
| 同步细纲 / 更新未来大纲 | Editor → `update_outline` |
| 重新生成未来细纲 | Editor → `generate_outline` |
| 改名 / 改一条规则 | 待直调 `update_field` |
| **同步设定 / 补摘要 / 扫描伏笔** | **只弹引导**，让用户点顶栏按钮 |
