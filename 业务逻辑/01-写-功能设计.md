## 1. 这篇管什么

| | |
|--|--|
| **作者想干什么** | 续写、重写、保存章节；开新书；从零生成设定 / 角色 / 关系 / 伏笔 / 地点 / 大纲 |
| **入口** | Chat · Creator；Editor 里保存正文、删章 |
| **铁律** | **写章和保存只动正文**。不生成摘要、不同步档案、不改未来细纲 |

## 2. 功能一览

### 2.1 正文（写完立刻落盘）

| 作者怎么说 | 做什么 | 代码 | 会写入 | 明确不做 |
|------------|--------|------|--------|---------------------|
| 续写 / 写下一章 | 生成并保存**下一未写章** | `continue_writing`（不传章号） | 正文 + 章索引 | 摘要、sync、细纲 |
| 写第 N 章 / 生成第 N 章 | 生成并保存**指定章**（须尚未写入） | `continue_writing`（`chapter_num=N`） | 同上 | 该章**已写过**则拒绝，提示用「重写」 |
| 重写第 N 章 | 覆盖指定章正文 | `regenerate_chapter` | 同上 | 同上 |
| 编辑器里改完保存 | 保存正文 | `chapter_service.update` | 正文 + hash | 正文变了则**清空**该章旧摘要 |
| 手动加章 | 新增章节文件 | `chapter_service.add` | 正文 + 索引 | 摘要可选，通常为空 |
| 删章 | 删文件和索引 | `chapter_service.delete` | — | 不调 LLM；摘要条目一并删 |
| 流式生成中断 | 恢复已生成的部分 | `_stream_chapter_content` | 每 200 字直接写入章节文件；下次续写自动从已有内容继续 | — |

### 章节元数据（ChapterOutline 字段）

写入/保存时系统自动记录的额外字段（[models.py#L22-L45](file:///d:/Novel-LangGraph/InkFlow/novel_agent/core/models.py#L22-L45)）：

| 字段 | 说明 | 自动写入？ |
|------|------|-----------|
| `pov_character` | 主视角角色名 | ❌ 手动填写（未来可 AI 推断） |
| `timeline_marker` | 时间线标记（如"第3天-上午"） | ❌ 手动填写 |
| `status` | 章节状态：draft / revised / final | ✅ 写章时自动设 draft |
| `word_count` | 字数统计 | ✅ 保存时自动计算 |
| `content_hash` | 正文 SHA-256 前 16 位 | ✅ 保存时自动计算 |
| `content_summary` | 章节摘要 | ❌ 同步设定时 LLM 生成 |

**这些字段不增加用户操作负担**——`word_count`、`content_hash`、`status` 全自动，`pov_character` 和 `timeline_marker` 是可选元数据，作者有需要才填。

### 2.2 开新书

| 作者怎么说 | 做什么 | 代码 | 会写入 |
|------------|--------|------|--------|
| 创建新书 | 初始化工作区 | `init_novel` | meta、空目录结构、可选首批空字段 |

### 2.3 从零生成设定（书还是空的，或某字段还没有）

| 作者怎么说 | 生成什么文件 | 代码 |
|------------|--------------|------|
| 生成设定 | `settings.md` | `generate_settings` |
| 生成角色 | `characters.md` | `generate_characters` |
| 生成关系 | `relationships.md` | `generate_relationships` |
| 生成伏笔清单 | `foreshadowing.md` | `generate_foreshadowing` |
| 生成地点表 | `locations.md` | `generate_locations` |
| 生成未来细纲 / 大纲 | `outline_future.md` | `generate_outline` |

**`generate_outline` 不限于新书：** 已有书说「重新生成未来细纲」也会走同一 handler（路由多在 [01-改](01-改-功能设计.md) · Editor）。与 **`update_outline`（同步细纲）** 的区别是：前者**整份重做**规划，后者在已有摘要基础上**增量**改后面几章打算。

**冷启动 vs 已有书：**

| 场景 | 走哪 | 说明 |
|------|------|------|
| 书还空 / 细纲文件空 | 写 · `generate_outline` | 开卷前搭规划 |
| 已有书「同步细纲」 | 改 · `update_outline` | 不动已写正文，只调打算 |
| 已有书「重新生成未来细纲」 | 改 · `generate_outline` | 整份覆盖 `outline_future.md` |

实现集中在 `novel_agent/agent/tools/generate.py`。

**和另外两册的区别：**

- 已有书改一条规则 → [01-改](01-改-功能设计.md) · `update_field`
- 正文写完后把新事实收进档案 → [01-对](01-对-功能设计.md) · `daily_sync`

## 3. 续写内部怎么跑

```
定章号/标题 → 流式生成正文 → 可选自检改写
→ 保存章节 → 更新 outline（已写、hash）
→ 章摘要留空 → 提示「摘要将在同步设定时更新」
```

续写时会读：未来细纲 + 已有档案 + 最近几章正文。档案若还没同步过，能写但容易偏。

**流式中断恢复：** 章节生成和字段生成（设定/角色/细纲等）均每 200 字直接写入目标文件。连接断开或页面关闭后，下次生成自动检测已有内容并从末尾继续，不从头开始。

## 4. 入口

### Chat

| 作者大概会说 | 调什么 |
|--------------|--------|
| 续写 / 写下一章 | `continue_writing`（自动找下一未写章） |
| 写第 N 章 / 生成第 N 章 | `continue_writing`（`chapter_num=N`；已写过则提示「重写第 N 章」） |
| 重写第 N 章 | `regenerate_chapter` |
| 创建新书 | `init_novel` |
| 生成设定 / 角色 / 关系 / 伏笔 / 地点 | 对应 `generate_*` |
| 生成未来细纲 / 生成大纲 | `generate_outline`（新书常见；已有书「重做细纲」见 01-改） |

路由表见 [索引 §7](01-意图分册索引.md)。

### Editor（不经过 Chat）

| 操作 | 调什么 |
|------|--------|
| 保存章节 Markdown | `chapter_service.update` |
| 删章 | `chapter_service.delete` |

保存后若正文有改动，旧摘要作废，等作者点 **[同步设定]** 再补。

## 6. 自测

| 操作 | 预期 |
|------|------|
| Chat「续写」 | 只写正文；章摘要仍空；不 sync |
| Chat「写第 5 章」/「生成第 5 章」 | 若第 5 章未写 → 写第 5 章；若已写 → 提示用重写 |
| Chat「重写第 5 章」 | 覆盖第 5 章正文 |
| 保存改正文 | 正文更新；该章摘要被清空 |
| 删章 | 正文和摘要条目都没了 |
| Chat「生成设定」 | 写出 settings.md；不 sync |
| 开新书 + 生成角色和大纲 | init → generate_characters / generate_outline |
