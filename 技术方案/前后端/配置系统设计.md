## 1. 总体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    配置系统架构                                │
│                                                               │
│  ┌─────────────────────┐  ┌─────────────────────┐           │
│  │  llm_config.json    │  │  token_config.json   │           │
│  │  LLM 模型/API 配置   │  │  截断/阈值参数配置    │           │
│  └──────────┬──────────┘  └──────────┬──────────┘           │
│             │                         │                      │
│  ┌──────────┴──────────┐  ┌──────────┴──────────┐           │
│  │  load_config()      │  │  get_truncation_config│          │
│  │  → dict             │  │  → TruncationConfig   │          │
│  └──────────┬──────────┘  └──────────┬──────────┘           │
│             │                         │                      │
│  ┌──────────┴──────────┐  ┌──────────┴──────────┐           │
│  │  llm.py             │  │  全局 tc 变量         │          │
│  │  DEFAULT_MODEL 等   │  │  from config import tc│          │
│  └─────────────────────┘  └─────────────────────┘           │
│                                                               │
│  ┌─────────────────────┐                                     │
│  │  WORKSPACE_DIR      │                                     │
│  │  工作空间路径配置     │                                     │
│  └─────────────────────┘                                     │
└─────────────────────────────────────────────────────────────┘
```

**设计原则**：
- 两套配置文件分离：LLM 配置（模型选择/API Key）与 Token 配置（截断阈值）独立管理
- 懒加载：配置路径在首次访问时解析，避免 import 时副作用
- 环境变量覆盖：支持通过环境变量指定配置文件路径
- 全局单例：`TruncationConfig` 通过 `_truncation_cache` 缓存，进程内只加载一次

## 2. LLM 配置

> 源码：`config/loader.py` → `load_config()` → `llm.py`

### 2.1 配置项

| 键 | 默认值 | 说明 |
|----|--------|------|
| `default_model` | `deepseek-v4-flash` | 默认模型（chat_stream 用） |
| `tool_call_model` | `deepseek-v4-flash` | Tool Calling 模型（ReAct 用） |
| `compression_model` | `deepseek-v4-flash` | 压缩/评估/摘要模型 |
| `context_window` | `131072` | 模型上下文窗口大小（128K） |
| `api_key` | 环境变量 `OPENAI_API_KEY` | API Key |
| `base_url` | 环境变量 `OPENAI_BASE_URL` | API Base URL |

### 2.2 使用方式

```python
# llm.py 启动时加载
_LLM_CONFIG = _load_config()
DEFAULT_MODEL = _LLM_CONFIG.get("default_model", "deepseek-v4-flash")
TOOL_CALL_MODEL = _LLM_CONFIG.get("tool_call_model", "deepseek-v4-flash")
COMPRESSION_MODEL = _LLM_CONFIG.get("compression_model", "deepseek-v4-flash")
CONTEXT_WINDOW = _LLM_CONFIG.get("context_window", 131072)
```

### 2.3 多模型路由

| 调用场景 | 模型 | 调用函数 |
|---------|------|---------|
| ReAct 工作流 | `TOOL_CALL_MODEL` | `chat_tools_stream()` |
| 章节生成/字段生成 | `DEFAULT_MODEL` | `chat_stream()` |
| 摘要/评估/记忆提炼 | `COMPRESSION_MODEL` | `chat()` |

## 3. Token 截断配置

> 源码：`config/loader.py` → `TruncationConfig` + `token_config.json`

### 3.1 TruncationConfig 数据类

```python
@dataclass
class TruncationConfig:
    memory_long_term_chars: int = 3000       # MEMORY.md 硬上限（默认）
    memory_chat_msg_chars: int = 300         # 单条对话消息截断长度
    memory_cache_ttl_seconds: float = 2.0    # 上下文缓存 TTL
    system_prompt_max_tokens: int = 12000    # 系统提示词最大 token
    system_prompt_fixed_tokens: int = 2500   # 系统提示词固定 token 开销
    memory_chat_rounds: int = 10             # 对话历史加载轮数
    chapter_recent_chars: int = 12000        # 最近章节内容截断长度
    subagent_summary_chars: int = 500        # Subagent 结果摘要长度
    subagent_tool_result_chars: int = 200    # Subagent tool_result 截断
    subagent_compress_msg_chars: int = 300   # Subagent 压缩消息截断
    subagent_compress_total_chars: int = 3000 # Subagent 压缩总长度
    subagent_compress_result_chars: int = 500 # Subagent 压缩结果截断
    memory_update_long_term_chars: int = 4000 # 记忆更新长期记忆截断
    compression_msg_chars: int = 150         # 压缩时单条消息截断
    compression_input_chars: int = 2000      # 压缩输入截断
    lead_plan_summary_chars: int = 100       # Lead Plan 摘要截断
    lead_metadata_chars: int = 200           # Lead 元数据截断
    handoff_task_chars: int = 100            # Handoff 任务描述截断
    handoff_summary_chars: int = 200         # Handoff 摘要截断
    graph_task_complete_chars: int = 200     # task_complete 结果截断
    dispatch_snippet_chars: int = 500        # dispatch 错误片段截断
    chapter_content_summary_chars: int = 200 # 章节内容摘要截断
    evaluator_result_chars: int = 200        # 评估器结果截断
    evaluator_agent_response_chars: int = 1500 # 评估器 Agent 响应截断
    nudge_interval: int = 5                  # nudge 提醒间隔（轮数）
```

### 3.2 token_config.json 覆盖

`token_config.json` 中的值会覆盖 `TruncationConfig` 的默认值。当前部署配置将 `memory_long_term_chars` 从默认 3000 提升到 5000：

```json
{
    "memory_long_term_chars": 5000,
    // ... 其他值与默认值相同
}
```

### 3.3 加载机制

```python
def get_truncation_config() -> TruncationConfig:
    global _truncation_cache
    if _truncation_cache is not None:
        return _truncation_cache
    cfg = {}  # 从 token_config.json 加载
    _truncation_cache = TruncationConfig(**{**_TRUNCATION_DEFAULTS, **cfg})
    return _truncation_cache
```

**合并策略**：`_TRUNCATION_DEFAULTS`（JSON 文件值）覆盖 `TruncationConfig` 默认值，`cfg`（JSON 文件实际值）再覆盖。即：`TruncationConfig 默认值 < _TRUNCATION_DEFAULTS < token_config.json`。

### 3.4 全局访问

```python
# config/__init__.py
tc = get_truncation_config()

# 使用
from ...config import tc
content = content[:tc.memory_long_term_chars]
```

## 4. 工作空间配置

### 4.1 WORKSPACE_DIR 解析优先级

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1 | `NOVEL_AGENT_WORKSPACE` 环境变量 | 自定义路径，自动创建目录 |
| 2 | `{project_root}/workspace` | 项目根目录下（存在 meta.json 或 pyproject.toml） |
| 3 | `{cwd}/workspace` | 当前工作目录下 |

### 4.2 工作空间结构

```
workspace/
├── {book_name}/
│   ├── meta.json              # 书籍元信息
│   ├── outline_structure.json # 大纲结构
│   ├── settings.md            # 设定
│   ├── characters.md          # 角色
│   ├── relationships.md       # 关系
│   ├── foreshadowing.md       # 伏笔
│   ├── outline_historical.md  # 已写大纲
│   ├── outline_future.md      # 未来大纲
│   ├── short_memory.md        # 短期记忆
│   ├── MEMORY.md              # 长期记忆
│   ├── chat.db                # 对话数据库
│   └── chapters/
│       ├── 001.md
│       ├── 002.md
│       └── ...
```

## 5. 配置加载优先级

### 5.1 llm_config.json

| 优先级 | 来源 | 环境变量 |
|--------|------|---------|
| 1 | `NOVEL_AGENT_CONFIG` 指定路径 | `NOVEL_AGENT_CONFIG` |
| 2 | `novel_agent/config/llm_config.json` | — |
| 3 | `{project_root}/llm_config.json` | — |
| 4 | `~/.novel-agent/config.json` | — |

### 5.2 token_config.json

| 优先级 | 来源 | 环境变量 |
|--------|------|---------|
| 1 | `NOVEL_AGENT_TOKEN_CONFIG` 指定路径 | `NOVEL_AGENT_TOKEN_CONFIG` |
| 2 | `novel_agent/config/token_config.json` | — |

### 5.3 懒加载

```python
class _LazyConfigPath:
    __slots__ = ("_path",)

    @property
    def value(self) -> Path:
        if self._path is None:
            self._path = _find_config_path()
        return self._path
```

`CONFIG_PATH` 是 `_LazyConfigPath` 实例，首次访问 `.value` 时才解析路径。
