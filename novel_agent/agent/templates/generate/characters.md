## 三层角色

### 核心角色（9 项）
姓名、身份、实力、性格、外貌、背景、动机、**角色弧光**、**角色秘密**
→ 当前阶段持续推动剧情、有独立弧线的角色

**角色弧光**：这个角色从故事开始到现在的内在变化轨迹——从什么状态到什么状态。长篇灵魂，没有弧光的角色是 NPC
**角色秘密**：角色未公开的信息——身世秘密 / 隐藏动机 / 不为人知的过去。长篇悬念来源，标注"读者已知"或"仅角色自知"

### 活跃配角（4 项）
姓名、身份（与核心角色关系）、实力、性格（一句话）
→ 近 5 章出场、无独立故事线；超 5 章未出场 → 退场

### 已退场角色（1 行）
角色名（退场原因，章号）

## 升降级规则

- 配角戏份加重、出现独立弧线 → 升级核心（补全 9 项，尤其弧光和秘密）
- 核心连续多章无重要戏份 → 降级活跃配角（精简 4 项）
- 死亡/离开/长期不出场 → 移入已退场，保留一行不删

## 弧光维护（长篇核心）

角色弧光不是一次性设定，而是**随章节演进**：
- 每次角色发生认知 / 价值观 / 关系网的重大变化时，更新弧光描述
- 格式：`起点状态 → [章N: 触发事件] → 当前状态`
- 示例：`信任所有人 → [章5: 被挚友背叛] → 开始怀疑他人 → [章12: 遇到无条件信任的人] → 学会辨别信任`

## 成长节点

核心角色可标注**成长节点**——触发弧光变化的关键章节：
- 格式：`章N：事件 → 变化方向`
- 用于后续写作时快速定位角色发展的关键转折

## 更新要求

1. 保留有效信息，不随意删改
2. 用户直接提供的角色信息 → 直接更新/新增
3. 矛盾时以最新章节为准；无变化维度保留原内容
4. **新增章节后**：检查出场角色是否有状态变化，有则更新弧光
5. Markdown 格式输出
