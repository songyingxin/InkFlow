## 检测任务

分析新增章节中哪些关系信息需要更新：

1. **关系状态变化**：出场角色间的关系是否有变化（同盟 / 敌对 / 暧昧等）
2. **关系演变时间线**：重要关系是否有新的转折点（更新演变轨迹）
3. **新关系建立**：是否有新的人物关系或势力关系需要登记
4. **关系中的误解/秘密**：是否有新的信息差产生 / 旧的误解被揭示
5. **势力关系变化**：势力间的联盟 / 对立是否有新发展
6. **势力秘密协议**：是否有新的暗流 / 秘密协议被揭示

## 输出格式

无变化时输出：`无变化`

有变化时只输出 SEARCH/REPLACE 补丁：

```
<<<<<<< SEARCH
现有内容中的精确片段
=======
更新后的内容
>>>>>>> REPLACE
```

可输出多个补丁块。每个 SEARCH 块必须是当前关系图谱中的**精确原文片段**。

## 原则

- **宁可漏报不可误报**——普通互动不算关系变化
- SEARCH 块必须是现有内容的精确片段，不要改写原文
- 只输出需要修改的部分，不要重新生成整个文件
- 演变时间线格式：`初始状态 → [章N: 事件] → 当前状态`
- 新关系登记：追加到对应维度（人物关系 / 势力关系）末尾
- 信息差标注：`读者已知` / `仅A自知` / `双方均不知`
- 如果关系图谱需要大改，提示用户使用 generate_relationships 整体重构
