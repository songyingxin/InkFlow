## 检测任务

分析新增章节中哪些角色信息需要更新：

1. **角色状态变化**：出场角色的位置 / 心境 / 实力 / 认知是否变化
2. **角色弧光推进**：核心角色是否有认知 / 价值观 / 关系网的重大变化（更新弧光描述和成长节点）
3. **新角色登场**：是否有需要登记的新角色（判断是核心还是配角）
4. **角色升降级**：配角戏份加重需升级核心 / 核心长期无戏份需降级
5. **角色退场**：是否有角色死亡 / 离开 / 长期不出场
6. **角色秘密揭示**：是否有角色秘密被公开或部分揭示（更新秘密标注）

## 输出格式

无变化时输出：`无变化`

有变化时只输出 SEARCH/REPLACE 补丁：

```
<<<<<<< SEARCH
现有内容中的精确片段
=======
更新后的内容
>>>>>>> REPLACE
```

可输出多个补丁块。每个 SEARCH 块必须是当前角色档案中的**精确原文片段**。

## 原则

- **宁可漏报不可误报**——普通对话和行为不算状态变化
- SEARCH 块必须是现有内容的精确片段，不要改写原文
- 只输出需要修改的部分，不要重新生成整个文件
- 弧光更新格式：`起点状态 → [章N: 触发事件] → 当前状态`
- 新角色登记：追加到对应层级（核心角色 / 活跃配角）末尾
- 如果角色档案需要大改（如多个角色弧光同时断裂），提示用户使用 generate_characters 整体重构
