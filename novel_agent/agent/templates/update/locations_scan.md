## 检测任务

分析新增章节中哪些地点信息需要更新：

1. **新地点登场**：可命名、可能再次出场的地点 → 主要地点表追加一行
2. **地点状态变化**：陷落、焚毁、废弃、更名 → 更新状态列或写入「改名与废弃」
3. **控制势力变化**：易手 → 更新控制势力列
4. **末次章**：本章仍出现的地点 → 更新末次章列
5. **路线信息**：明确写出旅途/距离 → 路线节追加（可选）
6. **次级场景**：一次性室内/小场景 → 次级场景节追加一行

## 输出格式

无变化时输出：`无变化`

有变化时只输出 SEARCH/REPLACE 补丁：

```
<<<<<<< SEARCH
现有内容中的精确片段
=======
更新后的内容
>>>>>>> REPLACE
```

可输出多个补丁块。每个 SEARCH 块必须是当前地点档案中的**精确原文片段**。

## 原则

- **宁可漏报不可误报**——普通叙事细节、未命名的「某客栈」不必登记
- 具体城池/关隘/山门由本文件维护；**不要**把新地点写进 settings
- SEARCH 块必须是现有内容的精确片段
- 新地点可追加到主要地点表末尾（SEARCH 匹配表头或最后一行，REPLACE 追加）
- 若需大改整张表，提示用户使用 generate_locations 整体重构
